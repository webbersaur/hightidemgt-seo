1. Critical Issues (Fix Immediately)

1a. Robots.txt has a typo in the Sitemap URL

The sitemap directive reads:

Sitemap: hhttps://hightidemgt.com/page-sitemap.xml

"hhttps" — double "h". Search engines cannot discover the sitemap. Must be https://.

1b. Multiple H1 Tags on Homepage & Services Page

  • Homepage: 2 H1s — "High Tide Commodities Management" + "Contact us"
  • Services: 2 H1s — "Our Job is to Make Your Job Easier" + "ELECTRONIC RECYCLING"

Each page should have exactly one H1 that contains the primary keyword.

1c. Nearly All Images Missing Alt Text

About Us11 (100%)
Contact22 (100%)

Only 1 image on the entire site has alt text (the HTCM globe logo). This is a major accessibility and SEO failure.

1d. "sample-page" Still in Sitemap

https://hightidemgt.com/sample-page/ (the WordPress default) is indexed. Should be noindexed or deleted.

1e. Affiliate Pages Exposed in Sitemap

5 affiliate-related pages are in the sitemap that likely shouldn't be indexed:

  • /become-an-affiliate/
  • /affiliate-tos-page/
  • /affiliate-reset-password/
  • /affiliate-portal/
  • /affiliate-login/

These should be noindexed or excluded from the sitemap.

2. High Priority (Significant SEO Impact)

2a. CLS Score is BAD (0.27)

  • Threshold: Good < 0.1, Bad > 0.25
  • Root cause: 11+ web fonts loading and causing layout shifts (Open Sans, Poppins, Roboto, FontAwesome, Divi modules font, affiliate plugin Poppins fonts)
  • Fix: Add font-display: swap declarations, preload critical fonts, reduce font count, consider size-adjust fallbacks

2b. Viewport Disables User Zooming

<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=0">

This is an accessibility violation (WCAG 1.4.4) and can negatively affect mobile SEO. Remove maximum-scale=1.0 and user-scalable=0.

2c. Thin Content on Key Pages

About Us and Contact are especially thin. Homepage should be expanded with more keyword-rich content.

2d. No LocalBusiness Schema

The site has generic Organization + Article schema from Rank Math, but no LocalBusiness schema. For a CT-based service company, this is critical for local SEO. Should include:

  • Business name, address (119 Montowese St, Branford, CT 06405)
  • Phone number(s)
  • Service area (Connecticut)
  • Business hours
  • Service types

2e. Schema Shows Article Type on All Pages

Every page (including Contact, About, Services) has @type: Article schema. These are not articles — they should be WebPage or Service type. The Article schema is misleading to search engines.

2f. Author Displayed as Email Address

Schema and Twitter meta show the author as chrishauman@gmail.com instead of a proper name. This looks unprofessional in search results and leaks a personal email.

3. Medium Priority (Optimization)

3a. Missing Service-Specific Pages

The sitemap shows dedicated service pages that are good:

  • /services/data-center-decommissioning/
  • /services/data-destruction/
  • /services/electronic-recycling/
  • /services/it-asset-disposition/

But these aren't linked from the main nav — the Services dropdown uses anchor links (#CONSULTING, #LOGISTICS, etc.) to jump within the single /services/ page. Consider linking to the dedicated pages in the nav for better crawlability and keyword targeting.

3b. No Contact Form on Contact Page

The Contact page has zero forms (0 <form> elements). It only lists phone numbers, email, and addresses. A contact form improves conversions and signals to Google that this is a legitimate contact page.

3c. Homepage Title Has HTML Entity in Schema

Schema shows &amp; instead of & in the headline:

"headline": "IT Asset Disposition Connecticut | Data Destruction &amp; ITAD"

3d. Open Graph Image is Same Across All Pages

Every page uses the same OG image (HTMC1200.jpg). Each page should ideally have a unique, relevant OG image for better social sharing.

3e. No External Authority Links on Homepage

The homepage has zero outbound links to authoritative industry sources (e.g., R2/RIOS certification bodies, EPA e-waste guidelines, NIST data destruction standards). These help establish topical authority.

3f. Content Has Typos/Grammar Issues

  • "assests" → "assets"
  • "liablities" → "liabilities"
  • "emlimnate" → "eliminate"
  • "We Help to Free Up Valuable Team Time but Handling" → "by Handling"

3g. Phone Number in H3 Tag (All Pages)

The phone number (203) 687-9370 is wrapped in an H3 on every page (appears to be in the header/nav). Phone numbers should not be heading tags.

4. Low Priority (Nice to Have)

4a. No Blog / Content Marketing

No blog posts exist. For ITAD/data destruction, blog content targeting long-tail keywords would help organic traffic:

  • "how to dispose of old servers in Connecticut"
  • "ITAD compliance requirements"
  • "data destruction certificate of destruction"

4b. Images Served as PNG/JPEG, Not WebP

Images are served as .png and .jpeg. Converting to WebP would improve load times.

4c. No Lazy Loading on Images

All images have loading="auto" — none use loading="lazy". Below-the-fold images should be lazy loaded.

4d. Missing Breadcrumbs on Service Sub-Pages

Schema includes BreadcrumbList on inner pages (good), but visible breadcrumbs would improve UX and internal linking.

4e. Affiliate Plugin Loading Fonts Site-Wide

The "Indeed Affiliate Pro" plugin loads Poppins font files (Medium + Bold) on every page, contributing to CLS and load time, even on non-affiliate pages.
